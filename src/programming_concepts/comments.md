# 注释


所有程序员都会致力于使他们的代码易于理解，但有时一些额外解释是必要的。在这些情况下，程序员会在他们的源代码中留下 *注释，comments*，编译器会忽略这些注释，但阅读源代码的人可能会发现有用。

下面就是个简单注释：

```rust
// hello, world
```

在 Rust 中，惯用的注释风格会以两个斜杠开始一条注释，而这条注释会一直持续到行尾。针对超出单个行的注释，咱们将需要在每行都包含 `//`，像下面这样：


```rust
// 所以我们在这里要做一些复杂的事情，长到我们需要
// 多行注释来完成！呼！希望这条注释能
// 解释发生了什么。
```

注释也可以放在包含代码行的末尾：

文件名：`src/main.rs`

```rust
fn main() {
    let lucky_number = 7; // 今天我感受到了好运
}
```

但咱们将更经常看到他们以下面这种格式使用，注释位于其注释代码上方的单独行中：


文件名：`src/main.rs`

```rust
fn main() {
    // 今日感到幸运
    let lucky_number = 7;
}
```

Rust 还有另一种注释，即文档注释，documentation comments，我们将在第 14 章 的 [“发布代码箱到 Crates.io”](../crates-io/publishing.md) 小节中讨论。
